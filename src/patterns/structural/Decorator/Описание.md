
### Структурные шаблоны проектирования (Structural)

**Декоратор (Decorator)**

**Назначение**

Динамически добавляет новую функциональность в экземпляры классов.

**Декоратор (Decorator)** — это структурный паттерн проектирования, который позволяет динамически добавлять объекту новые обязанности или функциональность, оборачивая его в объекты-«декораторы». При этом исходный объект не изменяется, а дополнительная логика «подмешивается» извне.

**Преимущества:**

* гибкость — возможность динамически добавлять функциональность;
* соблюдение принципа единственной ответственности (SRP);
* повторное использование — один декоратор может применяться к разным объектам.

**Недостатки:**

* усложнение структуры при большом количестве декораторов;
* сложность отладки из-за зависимости итогового поведения от порядка оборачивания;
* риск усложнения логики следования операций при избыточном использовании декораторов.

**Применение**

**Шаблон «Декоратор» используется, когда:**

* нужно добавлять обязанности объектам без создания подклассов;
* нужно гибко комбинировать поведение во время работы;
* нужно расширять функциональность класса прозрачным способом.

**Примеры**

Web Service Layer: Декораторы JSON и XML для REST сервисов (в этом случае, конечно, только один из них может быть разрешен).
