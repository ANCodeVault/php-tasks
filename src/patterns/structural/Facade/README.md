
### Структурные шаблоны проектирования (Structural)

**Фасад (Facade)**

**«Фасад» (англ. Facade)** — структурный шаблон проектирования, который предоставляет упрощённый интерфейс к сложной подсистеме. Он не скрывает функциональность подсистемы, а предоставляет удобный способ доступа к наиболее часто используемым функциям.

**Принцип работы**

Фасад действует как единая точка входа, которая координирует и делегирует запросы нескольким классам подсистемы. Некоторые особенности работы:

* Клиент взаимодействует только с классом фасада, а не с несколькими классами подсистемы.
* Фасад внутренне вызывает необходимые методы разных подсистем.
* Классы подсистемы не знают о существовании фасада и работают друг с другом напрямую.з интерфейс — зависимости внедряются через специальный метод, определённый в интерфейсе, который класс реализует для получения необходимого сервиса.

Фасад может иметь урезанный интерфейс, не имеющий 100% функциональности, которой можно достичь, используя сложную подсистему напрямую, но он предоставляет именно те функции, которые нужны клиенту, и скрывает все остальные. 

**Применение**

Паттерн «Фасад» используется в разных случаях, например:

* **Упрощение работы с сложной системой** с несколькими подсистемами. Например, в приложении для электронной коммерции управление заказами включает системы управления запасами, обработки платежей и уведомлений — фасад упрощает эти взаимодействия.
* **Уменьшение зависимостей** между клиентским кодом и сложными подсистемами. Например, в мультимедийном приложении различные подсистемы (сжатие видео, обработка аудио, обработка файлов) можно accessed через фасад, что снижает прямые зависимости от этих подсистем.
* **Скрытие деталей реализации** подсистем от клиента. Например, в системе домашней автоматизации фасад может скрыть детали взаимодействия с различными устройствами (освещением, термостатами, системами безопасности), предоставляя простой интерфейс для их управления.
* **Реализация слоёв** в многоуровневой архитектуре. Например, в веб-приложении фасад может использоваться для предоставления чистого интерфейса к слою бизнес-логики, скрывая сложности доступа к данным и обработки.
* **Управление устаревшим кодом** — паттерн помогает предоставить упрощённый интерфейс к устаревшему коду. Например, в банковском приложении фасад может предоставить современный интерфейс для взаимодействия с устаревшей основной банковской системой, скрывая старые API и форматы данных.
