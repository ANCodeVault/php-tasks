
### Поведенческие шаблоны проектирования (Behavioral)

**Цепочка Обязанностей (Chain Of Responsibilities)**

**Registry (реестр)** — это дизайн-паттерн, который используется в программной разработке для централизованного управления и доступа к общим объектам или экземплярам. Он позволяет создать единое хранилище, где объекты регистрируются с уникальными идентификаторами, а другие части программы могут получать к ним доступ через этот реестр.

**Назначение**

Построить цепочку объектов для обработки вызова в последовательном порядке. Если один объект не может справиться с вызовом, он делегирует вызов следующему в цепи и так далее.

**Примеры**

* фреймворк для записи журналов, где каждый элемент цепи самостоятельно принимает решение, что делать с сообщением для логирования.
* кеширование: первый объект является экземпляром, к примеру, интерфейса Memcached. Если запись в кеше отсутствует, вызов делегируется интерфейсу базы данных.
* фильтр спама
