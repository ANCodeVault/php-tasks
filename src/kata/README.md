
# Code Kata PHP для выполнения кодовых ката

Кодовая ката (Code Kata) — термин ввёл Дэйв Томас, соавтор книги «Прагматичный программист», вдохновляясь японской концепцией ката в боевых искусствах. Кодовая ката — это упражнение в программировании, которое помогает программисту оттачивать свои навыки посредством практики и повторения.

Важно фокусироваться на процессе, а не на конечном результате, так как код, написанный во время выполнения ката, обычно не используется в реальных проектах.

Такие упражнения часто используются для развития навыков тестирования (TDD — test-driven development) и улучшения рабочего процесса.

Помните, что ключ к освоению TDD и кодовых ката — это постоянная практика и самосовершенствование. Речь идёт не о достижении совершенства за одну ночь, а о постоянном прогрессе.

Полный цикл TDD состоит из следующих этапов:

1. Добавление теста (красный этап).
2. Запуск всех тестов и проверка неудачи (красный этап).
3. Внесение изменений (зелёный этап).
4. Запуск тестов и убедиться, что все они успешно выполняются (зелёный этап).
5. Устранить дублирование с помощью рефакторинга (третий этап).
6. Повторение цикла.

Основная идея TDD: цикл **«Красный — Зеленый — Рефакторинг»**.
